MVP

For the MVP we decided to use the Ant Framework so we could skip the frontend part and go straight into prototyping the solution and test it rapidly.

Core services

The crusial part of the Platform is the internal services module and APIs such as:

  • Feedback module that allows us to collect direct feedback from all pages in one place
  • GEO for the proper ATM and offices maps with routes, amount of cash, working hours etc
  • File storage for better file management. Each file is stored in the database so any part of the website or any app could fetch it from one URL and get instant updates if needed
  • Short URL service
  • AB-test module that allows our teams to perform proper testing of their pages or any parts of it
  • Newsfeed and articles database for quick and easy publications within our website or any other platform via XML, JSON, RSS formats
  • Personalization that allows the interface to update itself based on user actions and other data. The basic example: if the user entered the website with a search link to some product page and went to the main page, the main page will feature this product more for this user during the session

The UI

After validating the MVP I started to update the final design. At this stage I was responsible for the main components of the Platform such as file management module, feedback module, presets of common blocks, articles and FAQ database. Other parts were made by the team using established interface patterns.
